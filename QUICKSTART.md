# 🚀 QUICKSTART - Come Iniziare

## ⚡ Inizia in 30 Secondi

### Opzione 1: Apri Localmente (Consigliato)
```bash
# Su Windows
1. Scarica i file index.html, styles.css, script.js
2. Clicca destro su index.html
3. Scegli "Apri con" > Chrome/Firefox/Edge
4. Fatto! 🎉

# Su Mac/Linux
1. Scarica i file
2. Apri Terminal nella cartella
3. Esegui: python -m http.server 8000
4. Vai a http://localhost:8000
```

### Opzione 2: Apri Online
Carica i file su:
- **GitHub Pages** (gratuito)
- **Netlify** (drag & drop)
- **Vercel** (auto-deploy)

## 🎯 Come Usare per la Scuola

### Prima della Presentazione
1. Apri il sito da solo
2. Leggi il PRESENTAZIONE.md
3. Pratica a voce alta il discorso
4. Testa il sito su 2-3 browser diversi
5. Prepara risposte alle domande

### Durante la Presentazione
1. Mostra il sito dal vivo (non schermate statiche!)
2. Fai fare il quiz ai compagni
3. Coinvolgi con domande aperte
4. Mostra il grafico e discutilo
5. Domanda azioni concrete ai compagni

### Dopo la Presentazione
1. Condividi il link ai compagni
2. Invitali a fare il quiz
3. Discuti i risultati
4. Parla di azioni concrete
5. Monitora chi effettivamente cambia

## ❓ Domande Frequenti

### D: Posso modificare il sito?
**R:** Sì! È tuo. Leggi DOCUMENTAZIONE.md per istruzioni.

### D: Posso aggiungere sezioni?
**R:** Certo! Guarda DOCUMENTAZIONE.md nella sezione "Come modificare il sito".

### D: Il sito funziona offline?
**R:** Quasi. Chart.js viene caricato da CDN (internet richiesto per il grafico). Scarica Chart.js localmente se serve offline.

### D: Posso pubblicarlo online?
**R:** Sì! Usa GitHub Pages, Netlify, o Vercel (tutti gratuiti).

### D: Quanto tempo ci vuole per presentarlo?
**R:** 15-20 minuti. Leggi PRESENTAZIONE.md per timeline esatta.

### D: Cosa succede se cade internet durante la presentazione?
**R:** Il sito funziona lo stesso, tranne il grafico Chart.js. Prepara screenshot di backup!

### D: Posso condividerlo con i miei amici?
**R:** Assolutamente! Più persone lo vedono, meglio è. È per la scuola!

### D: È ottimizzato per mobile?
**R:** Sì, 100%. Teste su qualsiasi cellulare.

## 📞 Supporto

Se qualcosa non funziona:

1. **Apri il sito** con Chrome o Firefox
2. **Premi F12** (DevTools)
3. **Vedi la tab Console** per errori
4. **Leggi DOCUMENTAZIONE.md** sezione Troubleshooting
5. **Controlla che tutti i file siano nella stessa cartella**

**Errori comuni:**
- ❌ "styles.css not found" → Verifica che styles.css sia nella stessa cartella di index.html
- ❌ "Chart not defined" → Aspetta che la pagina carichi completamente
- ❌ Quiz non funziona → F12 > Console per verificare errori JavaScript
